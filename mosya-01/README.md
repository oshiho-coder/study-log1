# 模写サイト①：レシピサイト トップページ

## サイト概要
レシピサイトのトップページを題材に、HTML / CSS の基礎構造と
レイアウト実装を目的として模写制作を行いました。

## サイト種類
- レシピサイト

## ページ構成
- トップページのみ（1ページ）

## 対応端末
- PC / スマートフォン対応

## 使用技術
- HTML5
- CSS3
- Grid Layout
- Flexbox（一部UI配置）
- レスポンシブ対応（メディアクエリ）
  
## 実装内容
- メインビジュアルの全画面表示
- Gridを用いたレシピ画像の3カラムレイアウト
- ボタン・SNSリンクの配置

## 講師からの指摘
- `<div class="main">` ではなく、  
  `<header> <main> <footer>` を用いたセマンティックな構造にするよう指摘を受けた
- `p` タグと `<br>` の役割を混同しており、  
  段落と改行の使い分けを意識するよう指摘された
- セミコロン重複などの typo を指摘された
- `overflow: hidden;` は `img` 要素自体には効果がない点を指摘された
- メインビジュアル画像について、  
  「どの範囲を見せたいか」という意図が手本とずれている点を指摘された

## 詰まった点
- 見た目の再現を優先し、HTMLの構造的な正当性を後回しにしていた
- 横並びの実装を子要素の `width` 調整で行おうとしていた
- Grid / Flex / inline-block の役割を明確に区別できていなかった
- `vh` を使えば画像が自動的にレスポンシブになると誤解していた
- `object-fit: cover` と `overflow: hidden` の役割を整理できていなかった
- 画像が切れる原因を `img` 要素側だけで考えてしまっていた

## 学んだこと
- レイアウト（列数・並び）は親要素の責務であり、子要素はサイズと見せ方に集中させる
- Gridでは `grid-template-columns: repeat(3, 1fr)` により、列幅は自動管理できる
- `vh` は画面基準で高さを指定する単位であり、画像の伸縮とは無関係
- `object-fit: cover` による画像のトリミングは仕様であり、バグではない
- `overflow: hidden` は表示制御であり、画像サイズを変更するものではない
- inline-block はレイアウト用途ではなく、テキスト寄りの箱である
- Grid / Flex は「親がレイアウトを決める」ための仕組み
- レスポンシブ対応は、列数や配置を意図的に切り替えて初めて成立する
- HTMLは意味と構造、CSSは見た目という役割分担がある
- 同じ見た目でも、実装の質（壊れにくさ・拡張性）は大きく異なる
