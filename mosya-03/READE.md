# 模写サイト3：ブランドサイト/コーポレートサイト

## サイト概要
架空ブランド「Wooden Jewelry」のコーポレートサイトを題材に、  
HTML / CSS の基礎構造理解と Flexbox・レスポンシブ対応の習得を目的として模写コーディングを行った。

## サイト種類
- コーポレートサイト（ブランド紹介）

## ページ構成
- ジュエリーブランドの紹介ページ

## 対応端末
- PC
- スマートフォン（767px 以下）

## 使用技術
- HTML5
- CSS3
- Flexbox
- Google Fonts（Crimson Text）
- RESS（CSS Reset）

## 実装内容
- Flexbox を使用したヘッダー・コンテンツの横並びレイアウト
- CSSカスタムプロパティ（`--padding`）による余白管理
- `object-fit: cover` を用いたメインビジュアルのトリミング対応
- メディアクエリによる PC / SP レイアウト切り替え
- `order` を用いたレスポンシブ時の表示順制御

## 講師からの指摘
- HTMLでは `<ol>` を使用しているのに、CSSでは `ul` を指定していた  
  → 構造（HTML）とスタイル（CSS）の不一致
- CSSカスタムプロパティ定義時にセミコロンが抜けていた
- PC用とSP用で、異なるセレクタに対して width 指定をしていた  
  → 同じ要素を操作できていなかった
- レスポンシブ時のレイアウト変更を「画面幅」だけで考えていた

## 詰まった点
- ナビゲーションリストに `<ol>` と `<ul>` のどちらを使うべきか判断できていなかった
- PCとSPで同じ要素を制御しているつもりでも、実際は別セレクタを指定していた
- Flexboxの `order` が「並び替え専用プロパティ」であることを理解できていなかった
- CSSの小さな記述ミスが、全体の挙動に影響することを軽視していた

## 学んだこと
- HTMLは「意味・構造」、CSSは「見た目・制御」という役割分担がある
- レスポンシブ対応では「どの要素を操作しているか」を最優先で確認する必要がある
- Flexboxは横並びだけでなく、順序制御が本質的な機能である
- CSS変数は便利だが、記述ミスの影響範囲が大きい

## 今後の改善点
- class名の命名粒度を整理し、似た名前（`.content` / `.contents`）による混乱を防ぐ
- レスポンシブ対応前に、PCとSPで変わる点を先に洗い出す
- 実装前に HTML 構造を図で整理してから CSS を当てる
- 見た目の崩れを CSS だけで追わず、HTML構造から確認する癖をつける
