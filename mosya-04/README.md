# 模写サイト④：ポートフォリオサイト

## サイト概要
写真家・デザイナーのポートフォリオサイトを想定した模写コーディング。  
複数セクション構成（About / Works / News / Contact）を通して、  
Flexbox・Grid・レスポンシブ対応・フォームの構造理解を目的として制作しました。

## サイト種類
- ポートフォリオサイト

## ページ構成
- Header（ロゴ / ナビゲーション / SNSリンク）
- Main Visual（PC / SP 切り替え）
- About
- Works（画像グリッド）
- News（dlリスト）
- Contact（フォーム）
- Footer

## 対応端末
- PC
- スマートフォン（600px 以下）

## 使用技術
- HTML5
- CSS3
- Flexbox
- CSS Grid
- リセットCSS（ress）

## 実装内容
- Flexboxを用いたヘッダーおよびフォームのレイアウト制御
- CSS GridによるWorksセクションの画像グリッド配置
- PC / SPで表示画像を切り替えるクラス制御（`.pc` / `.sp`）
- `object-fit: cover` を用いたメインビジュアルのトリミング
- `dl / dt / dd` を用いたNewsリストの構造化
- メディアクエリによる縦並びレイアウトへの切り替え
- フォーム要素の基本スタイリングとホバー表現

## 講師からの指摘
- PCとSPで表示を切り替える要素について、HTML構造とCSS制御の対応関係を整理する必要がある
- FlexboxとGridを使用しているが、それぞれの役割を意識して設計すること
- Newsセクションの `dt / dd` の幅指定と、レスポンシブ時の挙動を整理すること
- フォームの横並びレイアウトは、SP時の表示崩れを事前に想定すること

## 詰まった点
- PC/SPで画像を切り替える際、どの要素を表示・非表示にするか整理できていなかった
- Gridを使っているが、「なぜGridを選ぶのか」を説明できない状態だった
- `dl` の横並びレイアウトで、SP時の再設計が後手に回った
- フォームの label と input の関係を構造的に捉えられていなかった

## 学んだこと
- Flexboxは並び方向や順序制御、Gridは全体配置に向いている
- レスポンシブ対応では、PCとSPで「変える要素」を先に決める必要がある
- HTML構造を理解していないと、CSS調整が場当たり的になる
- 見た目の崩れは、CSSより先にHTML構造を確認すべき

## 今後の改善点
- Grid / Flexbox を使用する前に、レイアウト構成を図で整理する
- PC/SPの差分を事前に洗い出し、CSS設計を先行させる
- フォームは入力しやすさを意識したUI設計を検討する
- class名を役割ベースで整理し、汎用的すぎる命名を避ける
