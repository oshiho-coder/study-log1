# 模写サイト⑤：ストアサイト

## サイト概要
雑誌、カタログ、アンティーク系の本をメインで扱う本屋のストアサイトを想定した模写コーディング。  
背景画像を用いたビジュアル表現と、複数セクション構成のページを通して  
Flexbox・レスポンシブ対応・レイヤー構造の理解を目的として制作した。

## サイト種類
ストアサイト（1ページ）

## ページ構成
- Main Visual（背景画像＋ロゴ）
- Title セクション
- Heading（Archive / New）
- Item（背景画像＋テキスト＋ボタン）
- Info（Catalog / Antique）
- Footer

## 対応端末
- PC
- スマートフォン（800px 以下）

## 使用技術
- HTML5
- CSS3
- Flexbox
- background-image
- absolute / relative
- メディアクエリ
- RESS（CSS Reset）

## 実装内容
- `background-image` を用いたメインビジュアルおよびセクション背景の実装
- Flexboxによるカードレイアウト（Heading / Info / Footer）
- `position: absolute` を用いた画像上テキストの重ね配置
- `order` を用いたレスポンシブ時の表示順制御
- セクション単位でのクラス分割（BEM風命名）
- hover 時の視覚的変化（opacity）

## 講師からの指摘
- Flexboxを使っているが、「どの要素同士を並べているか」の整理が不十分
- absolute配置の基準要素（relative）を意識せずに実装していた
- 背景画像（background-image）と img タグの役割の違いを理解できていなかった
- PC/SPで同じ要素を操作しているかの確認が甘かった
- クラス名は付いているが、役割単位で説明できない箇所があった

## 詰まった点
- テキストが画像の中央に配置される理由を構造的に説明できなかった
- Flexboxとorderの関係を「見た目」で判断していた
- レスポンシブ時の縦並びを、設計ではなく後付けで対応していた
- 背景画像と高さ指定（vh）の関係が曖昧だった

## 学んだこと
- background-image は装飾目的、imgタグは意味を持つ画像として使い分ける必要がある
- absolute配置は、必ず基準となる親要素（relative）を先に確認する
- レスポンシブ対応はCSS調整ではなく、HTML構造理解が前提になる
- クラス名は「見た目」ではなく「役割」で付けないと保守性が下がる

## 今後の改善点
- 各セクションについて、HTML構造とCSSの対応関係を事前に言語化する
- Flexboxを使う前に「横並びにしたい単位」を明確にする
- PC/SPの差分を実装前に洗い出す
- absoluteを使う場面では、必ず基準要素を明示する
