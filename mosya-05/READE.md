# 模写サイト⑤：ストアサイト

## サイト概要
雑誌・カタログ・アンティーク系の本を扱うストアサイトを想定した模写コーディング。  
背景画像を用いたビジュアル表現と、複数セクション構成を通して  
レイアウト設計とレイヤー構造の理解を目的として制作しました。

## サイト種類
- ストアサイト（1ページ構成）

## ページ構成
- Main Visual（背景画像＋ロゴ）
- Title セクション
- Heading（Archive / New）
- Item（背景画像＋テキスト＋ボタン）
- Info（Catalog / Antique）
- Footer

## 対応端末
- PC
- スマートフォン（800px 以下）

## 使用技術
- HTML5
- CSS3
- Flexbox
- レスポンシブ対応（メディアクエリ）
- リセットCSS（ress）

## 実装内容
- `background-image` を用いたメインビジュアルおよび各セクション背景の実装
- Flexboxによるカード・情報ブロックの横並びレイアウト
- `position: relative / absolute` を用いた画像上テキストの重ね配置
- `order` を用いたレスポンシブ時の表示順制御
- セクション単位でのクラス分割による構造整理
- hover時の視覚的変化（opacity）

## 講師からの指摘
- Flexboxを使用しているが、「どの要素同士を並べているか」の整理が不十分
- absolute配置の基準要素（relative）を意識して実装する必要がある
- 背景画像（background-image）と img タグの役割の違いを整理すること
- PC/SPで同じ要素を操作できているかを常に確認すること
- クラス名は付けているが、役割単位で説明できない箇所があった

## 詰まった点
- テキストが画像中央に配置される仕組みを構造的に説明できなかった
- Flexboxと`order`の関係を見た目だけで判断していた
- レスポンシブ時の縦並び対応を後付けで行ってしまった
- 背景画像と高さ指定（vh）の関係を整理できていなかった

## 学んだこと
- background-imageは装飾目的、imgタグは意味を持つ画像として使い分ける
- absolute配置では、基準となる親要素（relative）を必ず確認する
- レスポンシブ対応はCSS調整ではなく、HTML構造理解が前提になる
- クラス名は見た目ではなく役割で付けないと保守性が下がる

## 今後の改善点
- 各セクションについて、HTML構造とCSSの対応関係を事前に言語化する
- Flexboxを使う前に「横並びにしたい単位」を明確にする
- PC/SPの差分を実装前に洗い出す
- absoluteを使用する場面では、基準要素を明示する
