# 模写サイト⑥：アパレルブランド EC商品ページ

## サイト概要
アパレルブランドのECサイトを想定した商品詳細ページを模写制作しました。  
商品画像・商品情報・サイズ選択・サイズ表など、ECサイトに必要な要素を一通り実装しています。

## 制作目的
- Grid / Flexbox を使い分けたレイアウト設計の理解
- 商品ページ特有の情報構造を意識したマークアップ
- PC / SP 両方で破綻しないレイアウト構築

## ページ構成
- 商品ページのみ（1ページ）

## 対応端末
- PC / スマートフォン対応（レスポンシブ）

## 使用技術
- HTML5
- CSS3  
  - Flexbox  
  - Grid Layout
- メディアクエリによるレスポンシブ対応
- Google Fonts
- リセットCSS（ress）

## 実装内容
- Gridを用いた商品画像・商品情報の2カラムレイアウト
- Flexboxによるヘッダーナビゲーション配置
- select要素を用いたサイズ・数量選択UI
- テーブルを用いたサイズ表の実装
- 画面幅に応じたレイアウト切り替え（800px / 515px）

## 工夫した点（講師助言を反映）
- padding に `auto` を指定していた点を指摘され、  
  余白指定の正しい使い分け（margin / padding）を見直した
- CSSが意図せず反映されない原因として、  
  CSS読み込み順（head内の記述順）の重要性を理解した
- 表組みで `th` と `td` を混在させていたため、  
  見出しセルとデータセルの役割を整理して修正した
- レスポンシブ時に要素が重複・崩れていた箇所を洗い出し、  
  不要な指定を削除して整理した

## 詰まった点
- レイアウト作成時に細部から作り始めてしまい、  
  全体構造を把握できず手戻りが発生した
- Gridで大枠を分けた後の各要素の幅調整に時間がかかった

## 学んだこと
- コーディング前にレイアウト構成を図に起こすことで、  
  wrapper構成やクラス設計に迷いにくくなると実感した
- 見た目だけでなく、HTML構造とCSSの役割分担を意識する重要性を理解した

## 今後の改善点
- クラス命名に一貫性がない箇所があるため、命名規則を意識して整理したい
- JavaScriptを用いたカート処理や動的UIの理解を進めたい
